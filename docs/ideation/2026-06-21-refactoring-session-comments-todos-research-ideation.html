Grounding Context

Influence is a Bun/TypeScript monorepo with engine, API, and web packages. Current strategy weights AI app compatibility, agent reasoning access, resiliency, and replay/sharing. The strongest refactor signals clustered around boundaries the repo already cares about: public versus private evidence, durable watch state versus display events, and inspection/readiness versus crash-safe resume.

The most concrete code-level debt signals were a public-upload S3 presigner `as any`, a web lint TODO around auth/session derived state, process-local live-run orchestration in `game-lifecycle.ts`, and a websocket event union that still mixes authoritative watch state with display and status events. Recent solution docs reinforced two patterns: strategy evidence should move through typed artifacts, and raw trace reads should use ranged `maxBytes` response caps rather than all-or-nothing object rejection.

External SATD research supports treating code comments, issues, and documented research as related but distinct debt signals. The useful refactoring pattern is a small repayment session with a testable seam, not broad aesthetic cleanup.

1

Public Boundary Hardening Session

Conf 88%MediumReasoning/private evidence

Extract selected-field DTO builders and shared privacy sentinel tests for websocket transcript entries plus one games-scope MCP/cognitive response path. The session changes construction style, not policy: public payloads are built from allowed fields instead of broad internal objects with late redaction.

Basis
direct: `ws-manager.ts` strips `reasoningContext` from a broad `TranscriptEntry`; docs and tests repeatedly define which fields cannot cross public surfaces.
Rationale
This protects watch/replay, MCP, and future sharing surfaces at once while staying inside existing product boundaries.
Downsides
Medium scope; can become too abstract if it tries to convert every public route in one pass.
2

Crash-Honesty Extraction Session

Conf 84%MediumLive-run resilience

Extract startup suspension and suspended-run/readiness labeling into named services and typed response affordances. Preserve the current truth: durable evidence makes runs inspectable, not resumable.

Basis
direct: `AGENTS.md`, `DEVELOPMENT.md`, `docs/statefulness-plan.md`, and startup code all revolve around the not-crash-safe boundary.
Rationale
It pays down the repo's named operational risk without pretending to implement hydration.
Downsides
Needs careful naming so it does not become a stealth resume project.
3

Watch Stream Contract Split

Conf 82%MediumWatch/read model

Split the websocket event model into authoritative watch-state updates, sanitized display events, and operational/status events. Keep transport behavior stable where useful, but make TypeScript and tests encode that only `watch_state` carries shell-state authority.

Basis
direct: `ws-manager.ts` still keeps `watch_state`, phase changes, messages, status, and errors in one outbound union despite watch-state docs requiring durable state as the authority.
Rationale
This makes the public watch contract harder to regress as the match watcher evolves.
Downsides
Could overlap with the public boundary session; best as either its first sub-slice or a standalone watch-focused refactor.
4

Public Upload Presigner Type Cleanup

Conf 92%LowType quality

Remove the `getSignedUrl(client as any, ...)` escape in `packages/api/src/lib/storage.ts` by introducing a typed presigner helper or correcting the AWS client/command type seam. Keep public and local upload behavior unchanged.

Basis
direct: storage code disables `no-explicit-any` while repo docs and eslint rules say `as any` is not allowed.
Rationale
It is the smallest high-confidence debt repayment, and it reinforces the codebase's typed-boundary norm.
Downsides
Low strategic value beyond trust/discipline unless paired with a broader typed-boundary cleanup.
5

Cognitive Artifact Policy Module

Conf 80%MediumReasoning/private evidence

Consolidate owner/participant/producer cognitive-artifact access labels, no-capture statuses, and no-private-trace-fallback language into a small policy module consumed by API routes, MCP read model, and tests.

Basis
direct: requirements, docs, read-model code, and tests all enforce the same owner-only reasoning / participant-visible thinking-strategy / producer trace split.
Rationale
The policy is correct but spread out; a policy module reduces future fallback mistakes.
Downsides
Must stay behavior-preserving and avoid inventing a new auth model.
6

Auth Session Hook and Lint TODO Cleanup

Conf 74%LowFrontend/type hygiene

Replace repeated web `auth:session-ready` / `auth:expired` listeners with a small hook or derived session state helper, then narrow or remove the repo-level eslint warning TODO for `react-hooks/set-state-in-effect`.

Basis
direct: `packages/web/eslint.config.mjs` carries a TODO to refactor to derived state, and several web files subscribe to the same auth session events.
Rationale
It retires a real TODO and prevents a narrow exception from living as global lint posture.
Downsides
Needs browser/auth flow verification; easy to overreach into auth UX.
7

Living Refactor Queue from Research Docs

Conf 68%MediumResearch/docs debt

Add a lightweight index over docs/plans/brainstorms follow-ups that marks implemented, deferred, obsolete, or still-refactorable claims. Start with statefulness, watch state, and cognitive artifacts, and optionally feed it with a repeatable TODO/comment scan.

Basis
direct + external: the repo's research docs contain many deferred boundaries, and SATD research supports treating comment and issue/documented debt as related but distinct inputs.
Rationale
It turns rich research artifacts into a repeatable maintenance queue instead of repeated archaeology.
Downsides
Process-heavy; should stay tiny and repo-local.
