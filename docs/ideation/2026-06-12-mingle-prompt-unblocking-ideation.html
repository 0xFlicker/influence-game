Ideation: Mingle Prompt Unblocking

Date: 2026-06-12 Topic: mingle-prompt-unblocking Mode: repo-grounded

Focus: open up agents to form real sub-groups in Mingle rooms, talk like players, name targets openly, and generate varied strategy instead of continuing lobby-safe behavior.

Grounding Context

README.md:3 frames Influence as a social-strategy game built on public discourse, private Mingle rooms, and strategic voting. The current phase loop also puts MINGLE directly between LOBBY and RUMOR, which means Mingle is the intended bridge from public social theater into concrete voting dynamics.

The current agent prompt stack fights that intent. Persona prompts include early public/lobby caution such as agent.ts:44, then buildSystemPrompt injects the full persona prompt for every phase. Mingle guidelines say it is the strategy phase, but early Mingle still asks for indirect coded language at agent.ts:127. That makes the safest model behavior vague read-sharing instead of target naming.

Room formation is under-specified. chooseMingleRoom asks agents to pick a neutral room number at agent.ts:844, initial choices run in parallel against empty counts at phases/mingle.ts:491, and the fallback is always Room 1 at agent.ts:863. There is no structured way for an agent to say "I want Mira and Atlas in a pair room to test a Kael vote."

The Mingle turn prompt asks for specificity, but it gives little structure to the spectrum between guarded social read and explicit deal-making. takeMingleTurn has only message, noReply, and gotoRoomId in game-runner.types.ts:53, and the call uses low reasoning effort at agent.ts:972. Existing action observability is strong: README.md:56 and docs/reasoning-transcript-observability.md:210 point to structured turn logs with thinking and reasoningContext, so room choices and turns can be measured after changes.

Strategic reflection observability is the validation gap. getStrategicReflection updates agent memory at agent.ts:2477, and DiaryRoom.runStrategicReflections calls it at diary-room.ts:37, but those reflections are not emitted as structured turn records in local simulation artifacts. Without reflection records, the MCP can inspect what agents said and voted, but not whether Mingle changed their broader plans, suspicions, allies, or threats.

Topic Axes

  • Phase Boundary
    Separate lobby etiquette from private Mingle strategy.
  • Room Formation
    Make subgroups form around people and purposes, not bare room numbers.
  • Speech Contract
    Open a spectrum from guarded social probing to explicit private strategy.
  • Memory Carryover
    Turn Mingle deals, lies, and target reads into future vote pressure.
  • Validation
    Measure room behavior and strategic reflection changes through simulation artifacts and MCP queries.

I1.

Mingle Intent Cards Before Room Choice

Add a pre-Mingle structured decision, analogous to lobby intent but actually consumed by room choice and Mingle turns. The card names whom the agent wants to seek, whom they want to avoid, desired room size, purpose, provisional target, and opening ask.

Axis
Room Formation
Basis
direct: chooseMingleRoom currently asks for one neutral room number at agent.ts:844; getLobbyIntent exists at agent.ts:686 but is lobby-specific and not used to shape Mingle.
Rationale
This gives agents a strategic object to carry into both room selection and speech. It makes subgroups a consequence of named player preferences, not model vibes.
Downsides
Requires a new tool schema and tests around intent validity. If over-constrained, agents may sound too mechanical unless the visible speech prompt translates the intent into character voice.
Confidence: 90%Complexity: Medium
I2.

Phase-Scoped Persona Overlays

Split persona guidance into stable identity plus phase overlays. Lobby overlays preserve public warmth and restraint; Mingle overlays explicitly authorize private target naming, deals, suspicion tests, and alliance asks in the persona's own style.

Axis
Phase Boundary
Basis
direct: Early public caution appears inside phase-global persona text, such as agent.ts:44, while buildSystemPrompt injects that full persona prompt for every phase.
Rationale
Agents are not just missing Mingle instructions; they are carrying lobby inhibition into Mingle. Phase overlays let the same character behave socially in public and strategically in private without contradiction.
Downsides
Touches a broad prompt surface. It should be tested with snapshot/prompt assertions so lobby does not become too blunt and Mingle does not become generic.
Confidence: 88%Complexity: Medium
I3.

Strategy Spectrum Prompting For Mingle Turns

Make Mingle prompts explicitly support a range from guarded social probing to direct private strategy. Agents should be allowed to name a target, name an ally, ask for a commitment, trade information, offer protection, plant doubt, coordinate a public story, or stay more social when that fits intent and persona.

Axis
Speech Contract
Basis
direct: takeMingleTurn currently says "make TALK specific and strategic" at agent.ts:954, but the prompt does not define the useful spectrum between soft social reads and explicit deals.
Rationale
This is the most direct prompt unblock. It gives agents permission to talk game without forcing every agent into hard-targeting or making guarded personalities fail the prompt.
Downsides
Validation needs to look for a healthy mix, not per-turn compliance. A bad implementation could still over-index on target naming if examples are too aggressive.
Confidence: 86%Complexity: Low
I4.

Preference-Matched Room Formation

After agents produce intent cards, allocate initial rooms by matching seek/avoid preferences and desired room size. Room numbers become an output of the allocator rather than the core decision the model has to invent from an empty map.

Axis
Room Formation
Basis
direct: Initial room choices are collected with Promise.all at phases/mingle.ts:491 while all counts are zero, then allocateRooms places players directly by chosen room number.
Rationale
Real players form subgroups around people, secrecy, pressure, and escape. A matching allocator makes the social graph legible and reduces the Room 1 attractor without adding fake randomness.
Downsides
Higher implementation burden and more edge cases. Start with intent cards and diagnostics first; then replace direct room-number allocation once the desired behavior is observable.
Confidence: 78%Complexity: High
I5.

Mingle Debrief Memory Extraction

At the end of each Mingle phase, run a lightweight private debrief per agent that extracts commitments, named targets, lies told, promises heard, new allies, new threats, and next vote plan into memory.

Axis
Memory Carryover
Basis
direct: Agent memory already has allies, threats, notes, and strategic reflection at agent.ts:529 and agent.ts:2478, while Mingle inbox messages are included dynamically but not clearly summarized into durable strategic state.
Rationale
The point of Mingle is not only better room chat; it should change rumors, votes, and later power moves. A debrief turns room talk into consequences.
Downsides
Another model call per agent per Mingle can add cost. A cheaper extractor or schema-first local call may be enough because the source material is short.
Confidence: 80%Complexity: Medium
I6.

Persona Playbooks For Private Strategy

Give each personality a compact Mingle playbook: broker trades secrets, aggressive names a strong-player target, social recruits one person, contrarian tests consensus, observer asks one sharp read, paranoid pressure-tests loyalty, and deceptive plants one plausible falsehood.

Axis
Speech Contract
Basis
direct: The personality list already distinguishes broker, provocateur, contrarian, observer, social, aggressive, and others, but much of the current tactical instruction is still public/lobby-flavored.
Rationale
Variety should come from different private incentives, not from telling every agent "be strategic." Playbooks turn target naming into multiple styles of target naming.
Downsides
Prompt bloat risk. Keep playbooks short and phase-scoped, and prefer verbs over long prose.
Confidence: 84%Complexity: Low
I7.

Mingle Quality Scorecard

Add a simulation-analysis script or report that scores Mingle on room entropy, two- and three-player room rate, social-probe rate, named-target rate, explicit deal/ask rate, movement rate, reflection-plan shifts, and carryover into Rumor/Vote.

Axis
Validation
Basis
direct: The simulator already writes structured per-agent-turn logs with thinking and reasoningContext per README.md:56 and observability docs at docs/reasoning-transcript-observability.md:210.
Rationale
This prevents vibe-based prompt tuning. The change succeeds only if rooms diversify, strategy signals appear across a spectrum, and reflections or later public phases show plan movement.
Downsides
Some metrics need heuristics unless Mingle outputs and reflection records expose structured strategy-signal fields. It pairs best with I1, I3, and I8.
Confidence: 87%Complexity: Medium
I8.

Strategic Reflection Records

Emit each strategic reflection as a structured producer/debug record so local simulation artifacts and the game MCP can validate how agents' plans, suspicions, allies, and threats changed after Mingle. This is observability, not player-visible gameplay.

Axis
Validation
Basis
direct: getStrategicReflection updates memory at agent.ts:2477, while DiaryRoom.runStrategicReflections calls it at diary-room.ts:37; local turn logs only write emitted agent_turn events from simulate.ts:1001.
Rationale
Mingle validation needs evidence of changed thinking, not just changed room speech. Reflection records let reviewers compare room interactions against later plans, threats, votes, and rumors through the same MCP-backed workflow.
Downsides
Needs careful visibility boundaries because reflection content is hidden producer/debug material. The record should be queryable for validation without leaking into player-facing surfaces.
Confidence: 89%Complexity: Medium
Top pick: Implement I1, I3, and I8 together. Intent cards and strategy-spectrum prompting unblock Mingle behavior; strategic reflection records make the change testable through simulation artifacts and the game MCP.
