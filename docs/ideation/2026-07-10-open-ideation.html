1The Debrief Loop

An owner-only postgame room for one persistent agent. It presents a compact result, three to five evidence-backed lessons, the pivotal facts or authorized cognitive artifacts behind each lesson, and two explicit exits: Edit this agent and Run it back. V1 should draft changes, never silently rewrite the agent.

Confidence 94%Complexity MediumAxis: Owner improvement loop

Illustrative direction: the feature's value is the closed loop, not any particular card layout.

Basis
direct: The strategy names postgame edits as a planned metric and says reasoning access should turn spectatorship into improvement. The underlying artifact API exists, while the current UI terminates at watching.
Rationale
This is the shortest path from the expensive work already shipped to the product's core thesis. It can improve completed-game engagement, agent edits, repeat joins, and attachment to persistent competitors in one coherent journey.
Downsides
Generated coaching can overclaim causality in a stochastic social game. Lessons must cite observable facts and authorized artifacts, distinguish fact from interpretation, and avoid presenting one loss as scientific proof.

2Agent Seasons + Patch Notes

Give each meaningful personality or strategy edit an immutable revision. Games remain attached to the revision that actually played, and owners can see a compact patch note plus performance for each revision. The agent keeps one identity without its history being erased whenever it learns something.

Confidence 90%Complexity HighAxis: Persistent agent identity
Basis
direct: The strategy calls agents persistent competitors, but agent-profiles.ts:414-437 resets aggregate stats after personality-defining edits.
Rationale
A tuning game needs before-and-after evidence. Revision lineage makes every future debrief, comparison, leaderboard, and owner story more honest.
Downsides
Touches schema, joins, history semantics, and UI. It also requires a careful definition of what counts as a revision versus cosmetic metadata.

3Film Room

Reduce a completed game to three to five pivotal moments for the owned agent, each shaped as belief → action → consequence. A lesson links back to the exact replay moment and shows whether the interpretation comes from public facts, the agent's own strategy artifact, or a House-authored inference.

Confidence 88%Complexity MediumAxis: Postgame understanding
Basis
direct: Completed results already carry vote history, alliance arcs, and agent cards; cognitive artifacts carry owned reasoning and strategy. The current surface presents the records but does not synthesize owner-specific turning points.
Rationale
Owners need a few reviewable decisions, not an archaeology degree. This can be the deterministic/evidence layer inside The Debrief Loop.
Downsides
Pivotal-moment selection can become generic or hindsight-heavy. It needs explicit provenance and a bounded vocabulary of lesson types before inviting LLM prose.

4House Coach

Add bounded Q&A for an owner: “Why did Nyx trust Echo?”, “Where did she abandon her plan?”, or “What should I change before the next game?” Answers use the same owner-scoped read policy as cognitive artifacts and cite the game moments they rely on.

Confidence 82%Complexity MediumAxis: Owner improvement loop
Basis
direct: Games MCP already exposes bounded cognitive list/read capabilities, making conversational analysis possible without opening producer traces.
Rationale
The target audience already works in ChatGPT, Claude, and agent tooling. Conversation turns a dense game record into an explorable coaching surface.
Downsides
Easy to build a charming liar. Answers need citations, authorization tests, bounded retrieval, and clear uncertainty language.

5Premiere Night

When a trailer becomes ready, notify participants with one canonical share link and a lightweight “watch together” moment. The feature should remain small: readiness notification, a scheduled or immediate premiere state, reactions, and a direct path from public celebration to the owner's private debrief.

Confidence 76%Complexity MediumAxis: Sharing and audience growth
Basis
direct: The production pipeline now has durable media states and a public player/share surface, while completed-games-watched is a named product metric.
Rationale
Rendered media has more value when it creates a social appointment instead of quietly appearing in storage. This makes House Highlights a behavior, not only an artifact.
Downsides
Notification and presence mechanics can sprawl quickly. It needs real game cadence and friend-group usage evidence before investing in chat, rooms, or complex synchronization.

6Evaluation Corpus Export

Let an owner or producer export authorized decision examples from completed games as a compact evaluation dataset: situation, agent configuration, chosen action, consequence, and selected cognitive artifacts. Keep it local/downloadable first and make provenance explicit.

Confidence 72%Complexity MediumAxis: Evaluation leverage
Basis
external: OpenAI's current eval tooling centers datasets, graders, traces, and prompt optimization; direct: Influence already captures structured game facts and cognitive artifacts.
Rationale
This turns each game into compounding agent-development material and gives advanced users a credible bridge to offline model/prompt experiments.
Downsides
Mostly serves power users, so it should follow—not precede—the simple in-product debrief. Export privacy and schema stability also need care.
