Ideation

Multi-Round Strategy Propagation

The strongest direction is not to make every agent name a target. It is to let each agent carry a compact, evidence-aware private strategy across rounds, revise it when social evidence changes, and validate whether that private strategy actually influences later Mingle, vote, power, and council behavior. The 2026-06-13 extension adds a second continuity layer: a House-owned producer notebook that can name alliances, dramatize them, and feed sharper diary interviews and always-visible round summaries.

Grounding Context

The current engine already has a good single-phase strategic surface. At Mingle start, each agent forms a hidden intent; Mingle room choice and turn prompts receive a summary of that intent; private turn records carry mingle-intent, gotoPlayerName, and gotoStatus. Strategic reflection can also emit hidden records and updates lastReflection.

The gap is continuity. AgentMemory currently has allies, threats, notes, round history, power actions, and a most recent reflection, but no first-class strategy thread, commitment ledger, or evolving plan state. The memory store persists operational records during a game, yet the prompt-visible state is still mostly the live agent object. That means Mingle intent can guide the current room but does not automatically become durable social memory unless it turns into speech, notes, vote history, or a later reflection.

Existing project docs point in exactly this direction: end-of-Mingle debrief extraction, aggregate strategy scoring, and specialized strategy indexing were deferred follow-up work, while a hard requirement that every Mingle turn names a target was explicitly out of scope.

Design principle: carry forward strategy as revisable state, not orders. Agents should remember what they were trying to do, what evidence changed, and what social debts exist, while still allowing guarded, playful, social, or quiet play when that is strategically coherent.

1.

Strategy Thread / Carry-Forward Packet

Description: Give each agent a compact private strategy packet that persists across rounds: current objective, coalition posture, target posture, next Mingle ask, useful facts to test, promises to revisit, and abort triggers. Update it at reflection boundaries and inject it into later prompts as "live strategy context, not instructions."

Axis
Durable Strategy State
Basis
direct: getMingleIntent exists only as a pre-room-choice/phase context path, while AgentMemory contains no campaign/thread strategy object beyond lastReflection.
Rationale
This is the smallest object that makes a Mingle idea survive into the next vote, next room choice, or next power action without requiring agents to talk game every turn.
Downsides
Needs careful size limits and decay so agents do not cling to stale early reads. Badly phrased strategy packets could become orders instead of context.
Confidence 88% Complexity Medium
2.

Commitment Ledger

Description: Track promises, offers, requests, accusations, and claimed intentions as structured private memories. Each record should include speaker, audience, round, phase, evidence pointer, expected follow-up, status, confidence, and whether the outcome confirmed or contradicted it.

Axis
Commitments and Beliefs
Basis
external: Diplomacy-style social play depends on separating what someone promised, what the agent believes, and what later happened; prior agent-memory work also emphasizes reflection plus retrieval over raw transcript recall.
Rationale
This gives agents a memory of social debts. It lets strategy emerge from "Mira asked me not to expose Vex and then voted elsewhere" rather than generic suspicion lists.
Downsides
Commitment extraction can hallucinate intent from jokes or soft language. It needs evidence pointers and statuses such as proposed, accepted, violated, contradicted, expired, or unverified.
Confidence 84% Complexity Medium-High
3.

Reflection Delta Upgrade

Description: Upgrade strategic reflection from "current certainties/suspicions/allies/threats/plan" into an evidence-cited change log: what changed since the last reflection, what belief was weakened, what belief was strengthened, what promise needs follow-up, and what next experiment the agent should try. Keep the current summary fields, but add the delta so the next prompt can tell continuity from churn.

Axis
Reflection and Prompt Surfaces
Basis
direct: StrategicReflectionAction currently exposes snapshot fields, and getStrategicReflection stores only the summary without thinking or reasoningContext.
Rationale
Reflection is already the natural boundary for strategy propagation. Adding deltas makes it safer: agents can revise old reads instead of accumulating confident but unsupported narratives.
Downsides
More schema means more test and prompt maintenance. It also needs explicit contradiction handling so a reflection does not harden a false premise.
Confidence 82% Complexity Medium
4.

Strategy-to-Action Trace Validator

Description: Add a simulator/MCP validation query that answers whether private strategy propagated: intent shaped room choice, room behavior generated a signal, reflection incorporated or rejected it, and a later vote/power/Mingle action followed or consciously pivoted. Report examples, not just counts.

Axis
Validation and Observability
Basis
direct: Local evaluation docs already point users to game-N-turns.jsonl and MCP search_logs for mingle-intent, strategic-reflection, gotoPlayerName, and gotoStatus.
Rationale
This is how we avoid vibe-checking agent quality. The validation target becomes "show me a chain of strategic carryover or a clear pivot" instead of "did the prompt mention strategy?"
Downsides
Automated scoring will be fuzzy at first. The first version should present linked examples for human review before pretending to be a hard metric.
Confidence 86% Complexity Medium
5.

Hypothesis Portfolio

Description: Let agents carry two or three competing strategy hypotheses instead of one plan: primary read, alternate read, and chaos/unknown read. Each hypothesis gets confidence, evidence, next test, and switch condition. Later prompts can then ask the agent to choose, update, or abandon a hypothesis based on new observations.

Axis
Durable Strategy State
Basis
external: ReAct-style agents work best when reasoning, action, and observation interleave; in social deduction, premature certainty is often worse than uncertainty carried explicitly.
Rationale
This avoids the "reflection hardened a bad read" failure. Agents can pursue a line while staying live to contradictory evidence.
Downsides
More cognitive load in prompts. Small/local models may need tight formatting and a cap on active hypotheses.
Confidence 76% Complexity Medium
6.

Relationship Graph Memory

Description: Build a private per-agent relationship graph: trust, suspicion, usefulness, threat, debt, and volatility edges between the agent and other players. Update it from votes, power actions, Mingle commitments, and reflection deltas; summarize only the most relevant edges into prompts.

Axis
Commitments and Beliefs
Basis
external: Graph-informed social deduction is a strong hybrid pattern: structured belief tracking handles consistency while the LLM handles language and social framing.
Rationale
It gives strategy a substrate. Instead of "known allies: none" until a blunt alliance forms, the agent can remember soft social gradients like "useful but volatile" or "owes me a follow-up."
Downsides
Higher implementation burden and more room for noisy edge updates. Best as a second layer after the commitment ledger proves useful.
Confidence 70% Complexity High
7.

Resumable Strategy Hydration

Description: Make persisted memory recall part of agent setup or phase start so API games can hydrate strategy state after process reset. Start with reflection summaries, vote history, and future strategy packet records, then decide which fields are prompt-visible versus producer-only.

Axis
Persistence Boundary
Basis
direct: MemoryStore defines save, recall, and clear, but current prompt construction reads the live InfluenceAgent memory object.
Rationale
Multi-round strategy is less valuable if an API restart can wipe the live object and produce socially amnesiac agents. Hydration makes the strategy system operationally credible.
Downsides
Touches state ownership and reset/resume behavior, the known operational risk in this repo. It should follow the gameplay-memory design rather than lead it.
Confidence 74% Complexity High

2026-06-13 Extension: House MC Strategy Steward

This extension treats the House as its own strategic observer, not just a style wrapper over recent transcript text. Today GameRunner asks the House for freeform gameplay summaries after Mingle and Council using recent transcript slices. DiaryRoom asks the House for questions from recent public messages, player messages, and prior diary Q&A. Those are useful surfaces, but neither one gives the House a durable producer notebook that carries alliance hypotheses, faction names, contradictions, or unresolved story questions from one summary to the next.

The simulator shape is also important. In the current simulation config, diary interviews are effectively off because diaryRoomAfterPhases is set to an empty list. The --strategic-reflections flag turns on hidden reflection and Strategy Thread packet records; it does not by itself re-enable visible diary interviews. That suggests a clean design split: one option to include strategic reflection capture, and a separate simulation option or preset for actual diary sessions.

The explicit user direction for this pass is to spend context on the House. So the best ideas below do not optimize token footprint first. They prefer a broad House context pack, evidence pointers, and a private House dossier, then defer pruning until simulation quality or latency proves it necessary.

House principle: the House can name alliances before the agents do, but those names are producer/debug language until the product deliberately exposes them. The House should dramatize social structure without turning its read into canonical game truth.

H1.

House Strategic Dossier

Description: Give the House a private, round-to-round dossier that carries the producer read of the whole game: active alliances, named factions, rivalries, swing players, exposed contradictions, unresolved questions, likely next pressure points, and citations into recent transcript or turn artifacts. Refresh it before diary sessions and after each round boundary.

Axis
House-Owned Strategic Memory
Basis
direct: current generateGameplaySummary receives a recent transcript slice and returns a paragraph; there is no House-owned carry-forward object analogous to an agent Strategy Thread packet.
Rationale
This is the missing bridge between "the House saw a good moment" and "the House remembers that moment as part of the season arc." It lets diary questions and summaries build continuity without giving every player omniscience.
Downsides
It creates a second strategic memory system. The boundary must be explicit: House dossier state is producer/debug state, not canonical game state and not player memory.
Confidence 90% Complexity Medium-High
H2.

Alliance Naming Ledger

Description: Maintain a House-owned ledger of alliance and faction hypotheses. Each entry has a dramatic name, members, adjacent hangers-on, opposing faction, origin evidence, confidence, current status, last-seen round, and a short "why this matters" read. Names can be theatrical but must be tied to evidence.

Axis
Alliance Detection and Naming
Basis
reasoned: the House has global visibility across rooms, votes, public messages, and private producer artifacts; no single player has enough information to name the whole social topology safely.
Rationale
Named groups make the game watchable. "Mira and Kael keep orbiting each other while Vex brokers between them" is useful, but "The Velvet Pact is fraying because Vex keeps selling side deals" is the kind of continuity a viewer remembers.
Downsides
The House can over-name weak ties. Use confidence and status values like forming, active, strained, fractured, decoy, or retired so a colorful label does not harden into fact.
Confidence 86% Complexity Medium
H3.

Diary Pre-Interview Brief

Description: Before each diary session, ask the House to prepare a per-player interview brief from the dossier, current game state, recent messages, and that player's prior diary entries. The brief should choose one or two sharp angles: contradiction to probe, alliance pressure to name, promise to test, target posture to challenge, or story beat to pull into the open.

Axis
Diary Interview Prep
Basis
direct: DiaryRoomContext currently passes recent messages, player messages, and previous diary Q&A, but no House strategic packet, alliance ledger, or producer theory for the player.
Rationale
This makes diary rooms feel like the House has been paying attention for the whole season, not just the last few lines. It also lets diary ask better questions without forcing every agent answer to be a strategic confession.
Downsides
Extra House calls before every interview are expensive. For this feature that is acceptable initially, but the brief should be reusable across first question and follow-ups to avoid wasteful duplicate reasoning.
Confidence 88% Complexity Medium
H4.

Always-On Round Interstitial Summary

Description: Emit a House MC summary after every normal round and major endgame boundary, independent of --chatty. Keep --chatty as the live terminal stream, but always persist and print the round summary in the run artifacts / terminal progress so non-chatty simulations still have a narrative spine.

Axis
Always-On Narration
Basis
direct: current summaries are logged after Mingle and Council, while the simulation only prints full transcript entries live when chatty is true. Progress logs get structured transcript summaries, not a guaranteed House interstitial after each round.
Rationale
Simulation review should not require parsing a whole transcript just to know what happened. A round interstitial gives each batch a watchable recap and creates a stable place to refresh the House dossier.
Downsides
Can become noisy if every phase gets a summary. Prefer round-level boundaries by default, with phase-level summaries reserved for Mingle/Council or explicit diagnostic mode.
Confidence 84% Complexity Medium
H5.

Open-Pocketbook Context Pack

Description: For the first House MC upgrade, feed the House a deliberately broad context pack: recent transcript, relevant private turn records, latest Strategy Thread packet summaries, vote history, room assignments and movement, rumors, eliminations, prior House dossier, and prior diary highlights. Treat context size as a quality lever, not the first constraint.

Axis
House-Owned Strategic Memory
Basis
reasoned: the user explicitly accepted large House input contexts for this layer; the current failure is continuity and insight loss, not cost overrun.
Rationale
Optimizing prompt size before we know the right House read risks preserving the wrong abstraction. First prove that the House can produce better continuity; then compress around the fields that actually mattered.
Downsides
Large contexts can bury the signal and slow local runs. Add section headings and ranked evidence blocks so the House sees the important state before raw transcript volume.
Confidence 78% Complexity Medium
H6.

Drama-Safe Narration Grammar

Description: Give the House a narration grammar for alliance drama: distinguish confirmed events, House hypotheses, ironic framing, and open questions. It can say "the House suspects", "a quiet triangle is forming", or "this looks like a pact until the votes prove otherwise" instead of presenting every read as fact.

Axis
Evidence and Privacy Boundary
Basis
direct: current architecture already separates player-visible transcript, private agent_turn records, and canonical game events; House alliance labels should respect the same separation.
Rationale
This keeps the House fun without making it omniscient in the wrong way. The House can dramatize weak signals while preserving uncertainty and avoiding spoilers or fabricated certainty.
Downsides
If overdone, the summary can become hedgy. The grammar should still allow decisive prose when evidence is strong.
Confidence 82% Complexity Low-Medium
H7.

House Strategy Trace Validator

Description: Add a simulation/MCP review query that links House claims to evidence and later behavior. For example: named alliance entry -> cited room/vote evidence -> diary question generated from it -> agent answer -> later vote or Mingle movement that confirms, weakens, or contradicts the House read.

Axis
Evidence and Privacy Boundary
Basis
direct: current local evaluation docs already treat turns JSONL and MCP search_logs as the validation path for mingle-intent, strategy-packet, strategicLens, and packet-use markers.
Rationale
This prevents the House dossier from becoming vibes in a nice jacket. If the House names an alliance, reviewers should be able to inspect the evidence chain and see whether the read improved diary and summary quality.
Downsides
Do not start with a hard correctness score. Social reads are fuzzy; the first version should return linked examples, misses, and contradictions for human review.
Confidence 80% Complexity Medium
