Executive Takeaways

The rename should treat MINGLE as the canonical domain concept for new runtime behavior, while leaving WHISPER and whisper as legacy residue rather than supported current aliases. The central risk is not cosmetic copy; it is model-facing language that implies private whisper semantics during an open-room Mingle phase.

Follow-up requirements decision: Mingle is an entirely new current phase because it is better for the game. Do not purge old games, but do not build ceremony around supporting Whisper as a parallel phase. See docs/brainstorms/2026-06-11-mingle-phase-requirements.md.

Core Product Requirement

Agents, viewers, docs, simulator output, and current-game state should say Mingle. Historical records may remain, but polished old-game display is not required for the Mingle cutover.

Top Implementation Shape

Move the active engine, events, rows, prompts, and types to Mingle. Keep any Whisper handling narrow and local to historical artifacts that are still useful.

Detailed Requirement Set

R1. Canonical Domain Name

MINGLE is the canonical phase for new games, state machine transitions, logs, prompt context, display labels, and docs.

R2. Legacy Boundary Contract

WHISPER and whisper may remain in historical artifacts, but they are not supported current aliases for Mingle.

R3. Prompt Safety

New agent prompt text, tool names, system messages, and reasoning context must not say "Whisper" unless quoting historical data.

R4. Room Semantics

Mingle prompts should specify exactly who can hear a room message: private to current room occupants, not globally public, and not implied to be a one-to-one whisper.

R5. Config Aliasing

Current config keys use mingle. Legacy whisper keys are not part of the current game contract.

R6. Simulator Observability

--chatty, markdown summaries, simulation diagnostics, and local-model docs should use Mingle terms so reasoning review matches the current product.

R7. Frontend Replay Compatibility

Current games display Mingle. Old games that store WHISPER do not need polished display as part of this rename.

R8. Documentation Source of Truth

Root docs should define Mingle once and mark older Whisper-room docs as historical or superseded where they no longer describe live behavior.

R9. Regression Gate

A terminology check should distinguish allowed legacy references from accidental current-facing references.

R10. Reviewable Sequencing

Compatibility helpers and prompt changes should land before broad file renames so review can separate behavior from mechanical movement.

Compatibility Boundary

Data retention decision: do not purge old games as part of this rename. Existing rows may keep WHISPER phases, whisper scopes, and old room metadata. The current product contract moves forward: new runtime output emits Mingle vocabulary and old Whisper display is not guaranteed.

Allowed Legacy References

  • Narrow legacy guards and their tests.
  • Historical specs explicitly marked superseded.
  • Database migrations, replay fixtures, and old simulation fixtures.
  • External-facing release notes or docs that explain the rename history.

Data Retention Contract

  • Persisted historical transcript rows do not need to be rewritten or deleted.
  • New writes should use canonical Mingle phase/config vocabulary once the runtime rename lands.
  • Readers may tolerate old WHISPER values where needed to prevent unnecessary breakage, but should not advertise Whisper as a current phase.
  • A future cleanup can be considered only if old artifacts create real maintenance cost.
